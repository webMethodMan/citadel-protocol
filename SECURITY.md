# Security Policy

## Reporting Vulnerabilities

The Citadel Protocol is a high-assurance, reference architecture focused on hardware-enforced agentic governance using Trusted Execution Environments (TEEs) and cryptographic action permits. As this is an independent, non-commercial research project and reference implementation, I take security reporting seriously.

If you believe you have discovered a vulnerability in the `sakshi-core` logic or the cryptographic attestation loops:

### Reporting Process
Please submit your findings directly via email to:
**theo.ezell@gmail.com**

Please include:
*   **Description**: A clear summary of the potential security risk.
*   **Steps to Reproduce**: The specific environment, hardware requirements, or configuration needed to demonstrate the vulnerability.
*   **Impact**: Why this undermines the deterministic verification of the agentic action.

### Disclosure Policy
I am committed to resolving verified security vulnerabilities in a responsible manner. 
*   **Acknowledgment**: I will acknowledge receipt of your report within 72 hours.
*   **Investigation**: I will investigate the vulnerability to determine its impact on the Citadel Protocol’s integrity.
*   **Remediation**: If confirmed, I will provide a fix or architectural guidance in the repository as my time allows. 

**Note**: As this is an open-source reference implementation, please avoid public disclosure until we have had the opportunity to analyze and address the finding.
